# Настройка логирования

## Уровни логирования

В проекте используется система логирования с настраиваемыми уровнями. Уровни логирования (от самого детального к самому важному):

- `debug` - детальная отладочная информация
- `info` - общая информация о работе приложения
- `warn` - предупреждения
- `error` - ошибки

## Настройка в .env

Добавьте в ваш `.env` файл:

```env
# Logging Level (debug, info, warn, error)
LOG_LEVEL=info
```

## Доступные уровни

### debug
Включает детальное логирование:
- Raw данные всех входящих POST запросов к API
- Детальный анализ ошибок парсинга
- Полные заголовки запросов
- Stack trace для всех ошибок

### info (по умолчанию)
Стандартное логирование:
- Основные события приложения
- HTTP запросы с базовой информацией
- Ошибки без детального анализа

### warn
Только предупреждения и ошибки

### error
Только критические ошибки

## Примеры использования

### Для отладки проблем с API
```env
LOG_LEVEL=debug
```

### Для продакшена
```env
LOG_LEVEL=info
```

### Для мониторинга только ошибок
```env
LOG_LEVEL=error
```

## Что логируется в debug режиме

1. **Входящие запросы к API**:
   - Полные заголовки (кроме чувствительных)
   - Raw данные запроса
   - Content-Type и Content-Length
   - User-Agent

2. **Ошибки парсинга**:
   - Детальный анализ ошибок body-parser
   - Stack trace
   - Контекст запроса

3. **Необработанные ошибки**:
   - Полная информация об ошибке
   - Контекст запроса
   - Заголовки

## Безопасность

В debug режиме НЕ логируются чувствительные заголовки:
- `Authorization`
- `x-api-key`
- `Cookie`

## OpenSearch

Все логи отправляются в OpenSearch с соответствующим уровнем, что позволяет фильтровать их в Kibana или других инструментах визуализации.
